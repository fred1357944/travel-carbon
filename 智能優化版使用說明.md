### 2. 智能地理對應功能

- **新北市板橋區** → 智能識別為 **板橋** → **新北**

支援的新北地區：

- **浙江省杭州市** → 智能識別為 **杭州**
- **蘇州** → 自動對應到 **上海浦東機場**（最近的國際機場）
- **無錫** → 自動對應到 **上海浦東機場**

中國城市對應關係：
- 蘇州、無錫 → 上海機場

## 新增地點的方法

### 1. 新增台灣地區
在 COORDINATES 字典中添加：
```python
'新地區名': {'lat': 緯度, 'lon': 經度, 'name': '顯示名稱', 'type': 'domestic'},
```

如果是某個城市的區域，在 REGION_MAPPING 中添加：
```python
'區域名': '所屬城市',  # 例如：'天母': '台北'
```

### 2. 新增中國城市
在 COORDINATES 字典中添加機場：
```python
'城市名': {'lat': 機場緯度, 'lon': 機場經度, 'name': '機場名稱', 'type': 'international'},
```

在 CHINA_KEYWORDS 列表中添加城市名。

如果城市使用其他城市的機場，在 REGION_MAPPING 中添加：
```python
'小城市': '大城市',  # 例如：'昆山': '上海'
```

### 3. 新增國際地點
在 COORDINATES 字典中添加：
```python
'國家名': {'lat': 機場緯度, 'lon': 機場經度, 'name': '機場名稱', 'type': 'international'},
```

## 輸出說明

### Excel 檔案包含
1. **距離計算結果**工作表
   - 保留所有 1039 筆原始資料
   - 顯示智能對應後的地點名稱
   - 包含距離、時間等計算結果

2. **地圖**工作表
   - 每個唯一路線只產生一張地圖
   - 列出使用該路線的所有傳票編號
   - 地圖會根據距離自動調整縮放級別

### 結果分類
- **成功處理**：找到座標並計算距離的資料
- **無出差地點**：原始資料中沒有填寫出差地點
- **未知地點**：無法識別或找不到座標的地點

## 注意事項

1. **地名輸入建議**
   - 可以輸入完整地名（如「台北市南港區」）
   - 也可以只輸入關鍵地名（如「南港」）
   - 程式會自動處理並找到最合適的對應

2. **中國城市處理**
   - 所有中國城市都視為國際出差
   - 自動對應到最近的國際機場
   - 計算從桃園機場飛往目的地的距離

3. **快取機制**
   - 相同路線只計算一次
   - 下次執行時會使用快取結果
   - 可以使用「清除快取」按鈕重新計算

## 疑難排解

### 如果某個地點無法識別
1. 檢查是否有錯別字
2. 嘗試簡化地名（如只輸入城市名）
3. 查看終端機的警告訊息
4. 在程式碼中新增該地點的座標

### 如果要處理特定範圍
可以手動調整「起始列」和「結束列」，只處理部分資料。

### 查看處理進度
程式會顯示：
- 目前處理第幾筆資料
- 發現多少個不同目的地
- 快取命中率
- 處理完成統計
