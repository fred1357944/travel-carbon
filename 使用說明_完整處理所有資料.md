# 出差距離計算與地圖產生工具 - 使用說明

## 問題解決

### 為什麼只處理了 746 筆而不是 1039 筆？

原因：程式預設的處理範圍是第 1-10 列，所以只處理了部分資料。

**解決方案：**
1. 程式已經修改為自動處理所有資料
2. 載入資料後，「結束列」會自動設定為資料總筆數
3. 使用「一鍵處理全部」按鈕可以處理所有資料

## 主要改進

### 1. 自動處理所有資料
- 載入資料後會自動將處理範圍設定為全部資料
- 不再預設只處理前 10 筆

### 2. 處理沒有出差地點的資料
- 即使某些資料沒有出差地點，也會保留在結果中
- 這些資料會標記為「無出差地點」

### 3. 處理未知地點
- 如果地點不在座標資料庫中，會標記為「未知」
- 不會因為找不到地點而跳過整筆資料

### 4. 新增「一鍵處理全部」功能
- 自動依序執行：載入資料 → 計算距離 → 產生地圖 → 匯出 Excel
- 適合處理大量資料

## 使用步驟

### 方法一：手動逐步處理
1. **載入資料**
   - 點擊「瀏覽」選擇 Excel 檔案
   - 點擊「載入資料」
   - 確認顯示的資料筆數正確（應該是 1039 筆）

2. **計算距離**
   - 確認「結束列」顯示為總資料筆數（1039）
   - 點擊「計算距離」
   - 等待計算完成

3. **產生地圖**
   - 點擊「產生地圖」
   - 程式會為每個唯一路線產生地圖

4. **匯出 Excel**
   - 點擊「匯出Excel」
   - 結果會儲存在指定的輸出目錄

### 方法二：一鍵處理（推薦）
1. **載入資料**
   - 點擊「瀏覽」選擇 Excel 檔案
   - 點擊「載入資料」

2. **一鍵處理全部**
   - 點擊「一鍵處理全部」按鈕
   - 確認要處理所有資料
   - 程式會自動完成所有步驟

## 執行程式

```bash
# 方法 1：直接執行修改後的程式
python3 travel_distance_calculator_gui_cached_efficient.py

# 方法 2：使用啟動腳本
python3 launch_gui.py
```

## 預期結果

處理 1039 筆資料後，您應該會看到：
- **處理結果**：1039 筆（與原始資料相同）
- **成功處理**：有實際出差地點且找到座標的資料
- **無目的地**：沒有填寫出差地點的資料
- **未知地點**：有出差地點但不在座標資料庫中的資料

## 輸出檔案

Excel 檔案會包含：
1. **距離計算結果**工作表：包含所有 1039 筆資料的處理結果
2. **地圖**工作表：唯一路線的地圖截圖

## 注意事項

1. **座標資料庫**：程式內建了台灣各縣市和主要國際機場的座標。如果有新的地點，需要手動添加到 COORDINATES 字典中。

2. **處理時間**：處理 1039 筆資料可能需要幾分鐘，請耐心等待。

3. **快取機制**：程式會快取已計算的路線，相同路線不會重複計算。

4. **地圖產生**：只會為唯一路線產生地圖，相同路線共用一張地圖，提高效率。

## 疑難排解

### 如果程式沒有顯示視窗
```bash
# 使用 pythonw（macOS）
pythonw travel_distance_calculator_gui_cached_efficient.py

# 或使用強制顯示腳本
python3 force_gui.py
```

### 如果想要添加新的地點座標
在程式中找到 COORDINATES 字典，添加新地點：
```python
'新地點名稱': {'lat': 緯度, 'lon': 經度, 'name': '顯示名稱', 'type': 'domestic'},
```

### 如果要修改處理範圍
可以手動調整「起始列」和「結束列」的數值，只處理部分資料。
